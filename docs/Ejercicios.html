Ejercicio Herencia (interfaces, clases abstractas, implementar metodos)

  1. Crear interfaz Animal con un metodo "mueve" y "come"
  2. Crear clase abstracta Mamifero que implementa Animal y el metodo mueve (que pinte "El animal se mueve")
  3. Crear clase Reptil que implementa Animal y el metodo mueve
  4. Crear clase Perro que extienda Mamifero (que pinte "El perro se mueve") y que tenga un metodo propio "ladra" (que pinte "El perro ladra")
  5. Hacer un Main que ejecute los metodos de Mamifero, Perro y Reptil

Ejercicio Final Baraja (Objetos Java, metodos/atributos, switch, encapsulación)

  1. Crear clase carta con dos atributos: palo y numero
  2. Crear variable estatica con una lista de strings con los palos: "oros", "bastos", "espadas", "copas"
  3. Crear constructor de clase que acepte de entrada un palo y un numero (opcional, implementar validaciones)
  4. Crear un metodo getNumero que devuelva el numero salvo los palos especiales que devuelva el nombre: (1 -as, 11 -sota, 12-caballo, -13 - rey)
  5. Crear metodo toString (sobreescribe el toString por defecto) para que pinte la carta asi: "As de bastos", "5 de espadas", "sota de copas",...
  6. Crear un metodo equals que acepte de entrada una carta y devuelva si es la misma
  7. Crear un metodo getCuenta que acepte un ArrayList de cartas y devuelva la suma. Debera contar sota, caballo y rey como 10 (no como 11, 12, 13)
  8. Crear clase Baraja con un atributo ArrayList de cartas
  9. Crear constructor, sin parametros, que incializa las cartas creando todas las cartas posibles
  10. Crear metodo barajar que mezcla las cartas (Collections.shuffle)
  11. Crear metodo sacar que acepte un numero n y devuelva un ArrayList de n cartas (y las quite)
  12. Crear metodo sacar sin parametros y devuelva la primera carta (y la quite)
  13. Crear un Main que ejecute las siguientes acciones:
    • Crea una baraja
    • La mezcla
    • Saca 5 cartas
    • Muestra las cartas sacadas
    • Muestra la suma de las cartas sacadas
  14. Despues de mostrar las cartas, que pregunte al usuario si quiere mas y que siga hasta que diga que no o se quede sin cartas el mazo
